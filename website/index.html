Producto

Todo lo que necesita un BESS moderno

Desde la adquisición de datos hasta la ejecución de estrategias de arbitraje optimizadas con IA — en un solo sistema open source.

Optimizador MILP + DRL

Planificación de despacho por Programación Lineal Entera Mixta con fallback a agente DRL entrenado con datos CMg reales del nodo Maitencillo 220 kV (CEN/SEN Chile). Maximiza arbitraje respetando degradación de batería.

  • Solver HiGHS/CBC integrado
  • 48+ días de datos CMg históricos reales
  • Modelo de degradación semi-empírico (LFP, NMC, NCA)
  • Exportación ONNX para deploy edge (RPi 5)

Seguridad IEC 62443 SL-2

mTLS mutual auth, TOTP 2FA, rate limiting SR 7.1, bandit SAST, Trivy container scan y OpenSSF Scorecard en CI/CD. ~98% de cumplimiento.

WatchdogManager Autónomo

Autorecuperación sin intervención humana. Detecta desconexiones del driver y reconecta con backoff exponencial (1s → 30s). Métricas Prometheus integradas.

IEEE 2030.5 / SEP 2.0

Adaptador IEEE 2030.5 completo para cumplimiento CPUC Rule 21 y AEMO AS/NZS 4777.2. Compatible con aggregadores VPP y demand response.

Publicación Dual-Canal

GCP Pub/Sub + MQTT simultáneamente. Observabilidad con OpenTelemetry, métricas Prometheus en /metrics y health check en /health.

100% Open Source

Apache 2.0. Sin vendor lock-in. Certificación BESSAI disponible. Comunidad activa en GitHub con roadmap público y BEPs formales.

Arquitectura

Del dato al despacho en 5 pasos

01

Adquisición

Driver Modbus TCP lee SoC, potencia activa y temperatura cada 5 segundos desde el inversor real.

→
02

Seguridad

SafetyGuard valida rangos. Si algo sale del límite, bloquea la publicación y activa alertas inmediatas.

→
03

Optimización

MILP calcula el plan de despacho óptimo. DRL PPO lo refina usando precios CMg en tiempo real del CEN.

→
04

Publicación

Telemetría segura a GCP Pub/Sub + MQTT. Métricas Prometheus. Dashboard actualizado al instante.

→
05

Auto-sanación

WatchdogManager detecta fallos y reconecta sin intervención humana. El sistema sigue operando.
