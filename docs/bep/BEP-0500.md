## Motivación

Antes de BEP-0500, `FleetOrchestrator` y `VPPPublisher` existían como módulos aislados sin conexión entre sí. El fleet conocía su estado pero nunca podía actuar de forma autónoma sobre precios de mercado. BEP-0500 implementa `VPPFleetManager` como coordinador único.

---

## Loop de Control

```
1. FleetOrchestrator.run_cycle()      → FleetSummary
2. FleetOrchestrator.poll_all()       → SiteTelemetry por sitio
3. _select_strategy(precio, summary)  → DispatchStrategy
4. _compute_setpoints(tel, strategy)  → [SiteSetpoint], total_kw
5. _sync_vpp_sites(tel)               → VPPPublisher capacity update
6. VPPPublisher.publish_event(kw)     → OpenADREvent | None
7. return CycleResult
```

## Tabla de decisión de estrategia

| Precio (USD/MWh) | Fleet | Estrategia |
|---|---|---|
| ≥ 80 | Normal | `PRICE_ARBITRAGE` (discharge) |
| ≤ 40 | Normal | `PRICE_ARBITRAGE` (charge) |
