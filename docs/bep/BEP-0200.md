## Abstract

Este BEP define la arquitectura, interfaz y protocolo de integración del **Agente DRL de Arbitraje** (Deep Reinforcement Learning) para el BESSAI Edge Gateway. El agente reemplaza la estrategia rule-based con un modelo PPO/SAC entrenado con datos reales del mercado eléctrico chileno (CEN CMg), exportado a ONNX para inferencia en edge sin GPU.

**Meta de desempeño:** +25–35 % de ingresos respecto a la política rule-based baseline, validado sobre datos históricos CEN 2023-2025.

---

## Motivación

La política actual de despacho BESS es determinística (rule-based): carga cuando el precio es bajo, descarga cuando es alto, respetando límites de SOC y temperatura. Esta política:

1. **No anticipa** tendencias de mercado multi-hora.
2. **No aprende** de patrones estacionales o anomalías de precios.
3. **No optimiza** la degradación en función del valor marginal de cada ciclo.

Un agente DRL entrenado con datos históricos CEN puede resolver los tres problemas y generar +25-35% de ingresos adicionales documentados en literatura (Cao et al. 2020, Duan et al. 2022).

Negativo = carga · Positivo = descarga · 0 = idle.

### Función de recompensa

```python
def reward(soc, power_kw, cmg, prev_soh, curr_soh, is_safe):
    revenue        = power_kw * cmg / 1000        # USD por intervalo (5 min)
    degradation    = (prev_soh - curr_soh) * 1000 # penalización por degradación
    safety_penalty = 0.0 if is_safe else -10.0    # penalización hard por violación
    return revenue - degradation - safety_penalty
```

### Interfaz con `DataProvider`

El agente usa exclusivamente la interfaz `DataProvider`:

```python
# Observación
soc   = await driver.read_tag("soc_pct")
power = await driver.read_tag("active_power_kw")
temp  = await driver.read_tag("temp_c")

# Acción
await driver.write_tag("P_setpoint_kW", action * max_w_kw)
```

### Degradación graceful

```
DRL Agent (online)
    ↓ si falla o SOC out-of-range
ONNX Dispatcher (offline, último modelo)
    ↓ si falla
Rule-Based Policy (arbitrage_policy.py)
    ↓ si falla
Safety Guard determinístico (safety.py)
```
