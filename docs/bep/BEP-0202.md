| Campo | Valor |

| **Título** | Protocol Registry Unificado para Drivers BESS |
| **Autor** | BESSAI Engineering Team |
| **Estado** | 💡 Propuesto |

| **Supersede** | — |
| **Requiere** | BEP-0001 (proceso BEP), BESSAI-SPEC-001 (BESSDriver interface) |

## Motivación

El `Hardware Registry` actual (`registry/*.json`) cubre perfiles Modbus TCP de 4 fabricantes.
A medida que BESSAI se adopta en mercados internacionales, los protocolos de comunicación
se diversifican:

| Mercado | Protocolo dominante |
|---|---|
| Chile / América Latina | Modbus TCP + IEEE 2030.5 |
| EE.UU. (utilities) | DNP3 (IEEE 1815) + OpenFMB |
| Europa | IEC 61850 (GOOSE/MMS) + OPC-UA |
| Vehículos eléctricos (V2G) | OCPP 2.0.1 |
| Ecosistema LF Energy | Battery Data Format (BDF) |

El objetivo de BEP-0202 es definir un **Protocol Registry unificado** que permita
agregar soporte para nuevos protocolos sin modificar el núcleo del gateway.

## Diseño propuesto

### Abstracción: `ProtocolDriver` protocol

```python
# src/drivers/protocol_driver.py  (nuevo)
from typing import Protocol, Any

class ProtocolDriver(Protocol):
    """Interfaz unificada para todos los drivers de protocolo.
    
    Extiende BESSDriver (BESSAI-SPEC-001) con capacidades de
    protocolos industriales adicionales.
    """
    
    async def connect(self) -> None: ...
    async def disconnect(self) -> None: ...
    async def read_tag(self, tag: str) -> Any: ...
    async def write_tag(self, tag: str, value: Any) -> None: ...
    async def health_check(self) -> dict[str, Any]: ...
    
    # Extensiones BEP-0202
    @property
    def protocol_name(self) -> str: ...  # "modbus", "dnp3", "iec61850", etc.
    @property
    def protocol_version(self) -> str: ...
    async def subscribe(self, tags: list[str], callback: Any) -> None: ...  # para push protocols
```

### Formato del registro de protocolo

```json
{
  "protocol": "dnp3",
  "version": "IEEE-1815-2012",
  "transport": "TCP",
  "default_port": 20000,
  "driver_class": "src.drivers.dnp3_driver.DNP3Driver",
  "dependencies": ["pydnp3>=0.1.0"],
  "markets": ["US", "AU", "CA"],
  "standards": ["IEEE 1815", "IEC 60255-26"],
  "data_model": "BESSAI-SPEC-001"
}
```

## Compatibilidad hacia atrás

BEP-0202 es **no disruptivo**:
- El driver Modbus TCP existente seguirá funcionando sin cambios
- El env var `DRIVER_PROFILE_PATH` seguirá apuntando al perfil de dispositivo
- Se añade `DRIVER_PROTOCOL` (default: `modbus_tcp`) como nuevo env var opcional
