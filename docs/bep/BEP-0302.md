## Abstract

BEP-0302 define la integración de feeds de precios de mercado energético en tiempo real en el ciclo de decisión del `ONNXArbitrageAgent`. Se establece un `MarketDataClient` multi-fuente con fallback automático (CEN API v2 OAuth2 → Coordinador Excel scraper → caché local). El precio spot en vivo (`cmg_actual_usd_mwh`) reemplaza los datos históricos estáticos actualmente usados en el espacio de observación del agente DRL, transformando BESSAI en un sistema de **arbitraje energético dinámico** real.

---

## Motivation

El `ONNXArbitrageAgent` (BEP-0200) opera actualmente sobre datos CMg históricos de 48 días. Esto limita la capacidad de arbitraje a patrones pasados. En el mercado spot chileno (CEN / CDEC-SING), el CMg puede variar 300 % en minutos durante eventos de red. Un agente que responde al precio en tiempo real puede multiplicar su ventaja sobre la política rule-based.

**Benchmarks proyectados:**
- +15–25 % adicional de ingresos vs. BEP-0200 (datos históricos) al usar CMg en tiempo real
- Respuesta a picos de precio en < 30 segundos (vs. ciclo de control de 5 min con datos estáticos)

## Specification

### Arquitectura `MarketDataClient`

```
src/interfaces/market_data_client.py (NUEVO)

MarketDataClient
    │
    ├── CENAPIv2Source          ← OAuth2, endpoint /cmg/tiempo-real
    │   └── fallback (timeout 5s) →
    ├── CoordinadorExcelScraper ← Public Excel, actualización horaria
    │   └── fallback (stale > 2h) →
    └── LocalCacheSource        ← Último dato válido en memoria
```

```python
class MarketDataClient:
    """Multi-source CMg price feed with automatic fallback."""

    SOURCES = [CENAPIv2Source, CoordinadorExcelScraper, LocalCacheSource]

    async def get_cmg_live(self, node: str = "Maitencillo_220") -> CMgReading:
        for source in self.SOURCES:
            try:
                reading = await source.fetch(node, timeout=5.0)
                if reading.is_fresh(max_age_minutes=15):
                    return reading
            except MarketDataError:
                continue
        raise MarketDataUnavailableError("All sources failed")
```

### `CMgReading` — Tipo normalizado

```python
@dataclass
class CMgReading:
    node: str                    # "Maitencillo_220", "Quillota_220", etc.
    timestamp_utc: datetime
    cmg_usd_mwh: float           # Precio marginal en USD/MWh
    source: str                  # "cen_api_v2" | "coordinador_excel" | "local_cache"
    forecast_1h: float | None    # Predicción 1h del CMgPredictor v2
    forecast_4h: float | None    # Predicción 4h del CMgPredictor v2
    confidence: float            # [0.0, 1.0] — calidad de la fuente
```

### Integración con el agente DRL

El espacio de observación (BEP-0200, 8 variables) se enriquece:

| Variable | Antes (BEP-0200) | Ahora (BEP-0302) |
|---|---|---|
| `cmg_actual_usd_mwh` | Dato histórico estático 5-min | **Feed live < 15 min lag** |
| `cmg_forecast_1h` | CMgPredictor v2 sobre datos batch | **CMgPredictor v2 en tiempo real** |
| `cmg_forecast_4h` | Idem | **Idem, actualizado cada ciclo** |
| _(nuevo)_ `cmg_confidence` | — | Confidence del feed [0–1] |

### Nodos de mercado soportados (v1)

| Nodo CEN | Zona | Uso típico |
|---|---|---|
| `Maitencillo_220` | Zona Norte | Sitios Atacama / Coquimbo |
| `Quillota_220` | Zona Central | Chile Central |
| `Charrúa_220` | Zona Sur | Bio-Bío / Sur |
| `Pan de Azúcar_220` | Extremo Norte | SING legacy |

El nodo se configura via `.env`: `BESSAI_MARKET_NODE=Maitencillo_220`

### Requisitos normativos (MUST)

- Los datos de precios de mercado MUST ser publicados en telemetría (`/metrics`, OTel span) para auditoría.
- El sistema MUST operar en modo degradado (fallback a datos históricos) si todos los feeds fallan — MUST NOT detener el control de hardware.
- Las credenciales OAuth2 de la API CEN MUST almacenarse en variables de entorno (MUST NOT hardcodear).

## Backward Compatibility

- El `ArbitragePolicy` rule-based existente no es afectado.
- Si `MarketDataClient` no está configurado (vars de entorno ausentes), el agente continúa usando datos históricos del `CMgPredictor v2` — **sin regresión**.
- El endpoint `/api/v1/market` es nuevo — no rompe contratos existentes.

## Rejected Alternatives

| Alternativa | Razón de rechazo |
|---|---|
| Websocket directo al CEN | API pública no ofrece WebSocket — polling cada 5 min es suficiente para ciclo de 5 min |
| OpenADR 3.0 como única fuente | OpenADR pasa señales de demanda, no precios spot — ambos son complementarios |
| Hardcodear datos históricos para siempre | Elimina el 80 % del valor de arbitraje del agente DRL |
